# 项目功能
用户输入一段视频，这段视频被修改了人脸。
项目去寻找这段被修改后视频对应的在YouTube上的原视频，返回原视频的链接。

# 项目要求
使用python，uv，venv构建项目。使用docker部署，方便迁移使用。
使用LangChain/langgraph技术构建agent实现。
项目做成cli，要小而清晰。

# Workflow
初步的工作流：
读取被篡改后的视频片段。
按照一定间隔提取帧图片。
对提取的帧图做OCR，提取信息。
选择一张包含主要人物的，包含信息最多的帧图，调用serpapi的Google lens api进行搜索，获取到最相关的YouTube视频链接。
判断top N的YouTube视频链接哪个是本视频片段的原视频。
输出最可能的top 3视频链接给用户。

# 一些问题
OCR使用本地的开源的工具，包含全球语言。
调用serpapi的Google lens api可能无法从本地直接上传，可能需要图床等工具，提供一些好的解决方案供我选择。
获得YouTube链接后，如何比对是否是原视频，给我一些解决方案供我选择。
